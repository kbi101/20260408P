# 021: Live Trade Simulation & Ledger Specification

## Status: PROPOSED
**Owner:** QuantEdge Engineering
**Context:** Forward-Testing & Strategy Validation

## 1. Executive Summary
The "Live Trade Simulation" module provides an institutional-grade sandbox for executing and tracking recommendations generated by the Ensemble Engine. It enables users to validate "Live" signals in a simulated ledger, providing high-fidelity feedback on strategy performance before deploying actual capital.

## 2. Core Features

### 2.1 Recommendation-to-Trade Bridge
*   **Simulate Button:** Integrated into every `SwingRecommendationCard`.
*   **Auto-Capture:** Upon click, the system captures:
    *   `symbol`, `entry_price` (Last Price from recommendation), `entry_date` (Scan Date).
    *   `shares` (Hardcoded default: 100).
    *   `recommendation_id`: Link back to the parent signal.

### 2.2 Simulation Ledger (New Tab)
*   **Perspective:** A dedicated tab next to "Recommendations" in the Live Trading page.
*   **Live Tracker:** Displays all "OPEN" simulated positions with real-time (or latest available) P/L calculation.
*   **Historical Log:** Displays "CLOSED" positions for performance review.

### 2.3 Exit Orchestration
*   **Close Action:** A "Close Trade" button on each active transaction.
*   **Exit Capture:** Prompts or auto-fetches the latest price from QuestDB/YFinance.
*   **Journaling:** A markdown-supported `notes` field for recording the trade thesis or exit rationale.

## 3. Data Architecture

### 3.1 Schema: `quant.simulated_trades`
| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | SERIAL | Primary Key. |
| `symbol` | VARCHAR(10) | Ticker symbol. |
| `entry_date` | DATE | Date the signal was executed. |
| `entry_price` | DECIMAL | Execution price. |
| `shares` | INTEGER | Quantity (Default 100). |
| `exit_date` | DATE | Null if open. |
| `exit_price` | DECIMAL | Null if open. |
| `status` | VARCHAR(10) | `OPEN` or `CLOSED`. |
| `pnl_percent` | DECIMAL | Calculated on exit. |
| `notes` | TEXT | User comments/journal. |
| `recommendation_id` | INTEGER | FK to `ensemble_recommendations`. |

## 4. API Endpoints
*   `POST /api/v1/research/simulations/open`: Create a new trade.
*   `GET /api/v1/research/simulations`: Fetch ledger.
*   `PATCH /api/v1/research/simulations/close/{id}`: Close and journal a trade.

## 5. UI/UX Guidelines
*   **Visual Continuity:** The Simulation tab should match the "Scholar Ledger" aesthetic—clean tables, monospace figures, and glassmorphic accents.
*   **Feedback Loops:** Instant visual confirmation (Toast/Badge) when a recommendation is promoted to a simulation.
